## Scrum with UX — Dual-Track Agile (Security UX)

### Core Concept: Discovery + Delivery in Same Sprint
The Scrum Team balances two tracks within each Sprint:
- Discovery: research, design, experimentation, hypothesis validation
- Delivery: building, testing, deploying working Increments
Both tracks happen within the same Sprint, by the same cross-functional team.

### Security UX Principles
- MFA flows must be frictionless — don't create friction that causes users to bypass security
- Privacy settings easily accessible — not buried in menus
- Session timeout warnings visible (2 min before expiry)
- Confirmation dialogs before irreversible financial actions
- Security indicators visible (lock icons, HTTPS badge)
- Error messages never expose internal system details

### UX Anti-Patterns to Avoid
- Separate UX team doing design upfront, then "handing off" to Developers
- UX designer working alone, not part of the Scrum Team
- No user research — building features based on assumptions
- "Pixel-perfect" designs required before development starts (waterfall thinking)

### UX in Scrum Events
- Sprint Planning: include discovery work (experiments, research) in Sprint Backlog
- Sprint Review: present user feedback on both discovery findings and delivered Increments
- Retrospective: inspect how well the team balanced discovery and delivery
- Product Backlog: includes both discovery work and delivery work — not separate backlogs
