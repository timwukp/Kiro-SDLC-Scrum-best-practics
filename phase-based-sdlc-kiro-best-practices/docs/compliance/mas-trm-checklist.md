# MAS TRM 2023 Compliance Checklist

## 5. IT Project Management and Security-by-Design
- [ ] 5.1.1 Security requirements defined at project inception
- [ ] 5.1.2 Threat modeling completed for new features
- [ ] 5.2.1 Secure coding standards documented and enforced

## 6. Software Application Development and Management
- [ ] 6.1.1 Source code review process established
- [ ] 6.1.2 Static application security testing (SAST) integrated
- [ ] 6.2.1 Test environments segregated from production
- [ ] 6.3.1 Change management process documented

## 7. IT Service Management
- [ ] 7.1.1 Incident management procedures documented
- [ ] 7.2.1 Problem management process established
- [ ] 7.3.1 IT service continuity plan tested annually

## 8. Cyber Security Operations
- [ ] 8.1.1 Security monitoring implemented (SIEM/SOC)
- [ ] 8.2.1 Vulnerability management program active
- [ ] 8.3.1 Penetration testing conducted annually
- [ ] 8.4.1 Security incident response plan documented

## 9. Technology Risk Governance
- [ ] 9.1.1 Technology risk management framework established
- [ ] 9.2.1 Board oversight of technology risks
- [ ] 9.3.1 Third-party risk assessment for all vendors

## 10. Data and Infrastructure Security
- [ ] 10.1.1 Data classification scheme implemented
- [ ] 10.2.1 Encryption standards (AES-256 at rest, TLS 1.3 in transit)
- [ ] 10.3.1 Network segmentation implemented
- [ ] 10.4.1 Access control based on least privilege
